把hexo博客部署到服务器上

然后使用的宝塔面板，B站教程+deepseek操作，实现步骤后面有截图记录

实现hexo博客部署到服务器，核心是让用户通过网络访问服务器上的静态文件

服务器知识

公网

公网是 “互联网” 本身，是连接全球所有设备的公共网络，我们日常上网、访问网站、使用 APP，本质上都是在与公网中的设备交互
开放性，唯一性，全球性

IPv4 地址 由 4 组数字组成（如123.45.67.89），总数量约 43 亿个，已基本耗尽。 你的腾讯云服务器的公网 IP 就是 IPv4 地址，用户通过这个 IP 访问你的博客。
IPv6 地址 由 8 组字母 + 数字组成（如2001:0db8:85a3:0000:0000:8a2e:0370:7334），数量几乎无限（用于解决 IPv4 耗尽问题）

服务器

服务器本质是一台 “永远在线的高性能电脑”，用于存储数据、运行程序，并通过网络为用户提供服务

云服务器（ECS）

云服务器是物理服务器中分割出的 “虚拟电脑”，无需关心硬件，通过网络远程管理，按需付费

云服务器术语

  • 实例（Instance）
    云服务器的 “虚拟电脑” 本体，包含 CPU、内存、操作系统等核心资源
    在腾讯云购买的 “服务器” 就是一个实例
  • 镜像（Image）
    服务器的 “操作系统模板”，包含预装的 OS 和基础软件
  • 带宽（Bandwidth）
    服务器与互联网的 “数据传输通道宽度”，决定访问速度
  • 云硬盘（CBS）
    服务器的 “硬盘”，用于存储数据（如操作系统文件、Hexo 博客静态文件）
  • 公网 IP
    服务器在互联网中的 “身份证”，用户通过公网 IP 访问你的博客
  • 安全组（Security Group）
    服务器的 “防火墙规则集”，控制哪些端口 / IP 可以访问服务器

端口与域名

“端口” 和 “域名” 是服务器部署中最核心的两个 “导航元素”：端口决定数据从服务器的哪个 “门” 进出，域名决定用户用什么 “地址” 找到你的服务器

端口

端口：80（HTTP网页）443（HTTPS加密）8888（宝塔面板）22（SSH 协议）

理解：想象你的服务器是一栋大楼，里面有很多房间（服务）：博客在 80 号房间，加密博客在 443 号房间，宝塔面板在 8888 号房间，SSH 管理在 22 号房间。端口就是每个房间的 “门牌号”，用于区分不同的服务，确保数据能准确送到对应的程序。
为什么需要端口
服务器上可能同时运行多个服务（比如你的 Hexo 博客、宝塔面板、未来可能加的其他程序），每个服务都需要通过特定端口与外界通信。如果没有端口，数据到达服务器后就不知道该交给哪个程序处理

安全组

安全组相当于大楼的 “门卫”，负责控制哪些端口允许外部访问
添加入站规则（外部设备访问服务器的规则）：
允许0.0.0.0/0访问 80 端口（HTTP）：所有人能访问博客。
允许0.0.0.0/0访问 443 端口（HTTPS）：所有人能访问加密博客。
允许你的本地公网IP访问 22 端口（用于SSH管理）：只有你能远程管理服务器（如何查自己的公网 IP？百度搜索 “我的 IP” 即可）。
允许你的本地公网IP访问 8888 端口（宝塔）：只有你能登录宝塔面板。

域名解析

把域名解析到服务器公网的IP地址
主机记录 记录类型 记录值
@ A 服务器IP （@代表 “主域名”，比如你的域名是xxx.com，这条配置表示xxx.com指向你的服务器 IP。用户访问xxx.com会打开博客）
www A 服务器IP（www是子域名前缀，这条配置表示www.xxx.com指向你的服务器 IP。用户访问www.xxx.com也会打开博客。）

  • 这样配置后，用户无论输入xxx.com还是www.xxx.com，都会被引导到你的服务器，看到你的博客
  • A 记录：最常用的解析类型，作用是 “把域名和服务器 IP 绑定”，告诉互联网 “这个域名对应的 IP 是 xxx”

为什么需要域名？
好记.正规，可扩展性：未来如果更换服务器（IP 变了），只需修改域名解析，用户不用记新IP

为什么选Linux

服务器的 “操作系统” 是管理硬件和软件的核心，就像电脑的 Windows/macOS。但服务器几乎都用Linux，而非 Windows，原因如下：
开源免费：无需支付版权费用（Windows Server 需付费）。
稳定高效：Linux 对内存和 CPU 的占用更低，可长时间运行（数月不重启）。
安全性强：权限管理严格，病毒和漏洞远少于 Windows。
兼容性好：主流 Web 服务器（Nginx/Apache）、编程语言（Python/Node.js）更适配 Linux。
如何操作 Linux 服务器:
Linux 服务器通常没有图形界面，需通过SSH（远程登录协议） 用命令行操作

宝塔面板

宝塔：一个图形化的服务器管理工具，就像Windows的操作界面
替代复杂的Linux命令，通过网页界面管理服务器
主要功能：建网站、管理文件、安装软件、监控服务器
Nginx：网站的服务员，处理访问请求
FTP：文件传输工具(上传文件到服务器)

SSH

作用：远程登录，安全加密，文件传输
SSH：你通过宝塔面板管理服务器时，宝塔实际上是通过 SSH 在后台执行命令（比如安装 Nginx、创建网站）。若遇到宝塔无法解决的问题（如 Nginx 配置错误），需通过 SSH 手动修复

nginx

开源的高性能 Web 服务器、反向代理服务器及电子邮件代理服务器
Nginx：你的 Hexo 博客静态文件由 Nginx 托管，当用户访问你的域名时，Nginx 读取对应的 HTML/CSS/JS 文件并返回给浏览器
